# Créer une application {#create-an-app}

Pour ajouter une application à votre site, vous devez d’abord créer une application dans Livefyre Studio.

Pour créer une application :

1. Cliquez sur **[!UICONTROL +New App]** dans la page Applications, puis sélectionnez un type d’application dans le menu déroulant.
1. La boîte de dialogue **[!UICONTROL New App]** s&#39;affiche. Saisissez un **[!UICONTROL App Title]**, sélectionnez un **[!UICONTROL Site]**, puis cliquez sur **[!UICONTROL Create]** pour ouvrir la page Aperçu de l’application.

La page Résumé de l’application comporte 4 onglets :

* **[!UICONTROL App Summary]**: fournit un aperçu du contenu et de l’état de l’application, ainsi que des outils permettant d’accéder à l’application sur son site et de générer son code incorporé.
* **[!UICONTROL Designer]**: vous permet de sélectionner des options de conception pour l’application, notamment des polices, des modèles de couleurs et des options de configuration spécifiques à l’application.
* **[!UICONTROL Streams]**: liste toutes les sources de diffusion en continu pour le contenu de l’application. Cliquez sur le titre du flux pour ouvrir la page Détails du flux, à partir de laquelle il peut être modifié.
* **[!UICONTROL Settings]**: vous permet de sélectionner des paramètres spécifiques à l’application, notamment des règles d’ajout de contenu à l’application et les types de contenu à afficher. Cliquez sur l&#39;onglet **[!UICONTROL Settings]** pour remplacer les paramètres réseau et site de l&#39;application.

## Liste d’applications {#section_eyp_ykt_bbb}

La page liste Applications permet d’accéder à toutes vos applications Livefyre existantes.

La page liste Applications affiche les informations suivantes :

* **[!UICONTROL Title]**: Liste le titre de votre application et inclut une icône indiquant le type d’application.
* **[!UICONTROL Site]**: Liste l’emplacement de l’application, par site.
* **[!UICONTROL Content Count]**: Liste le nombre d’éléments de contenu publiés dans l’application.
* **[!UICONTROL Date Modified]**: Liste la dernière fois que l’application a été modifiée dans App Designer.
* **[!UICONTROL Embed Code (</>)]**: Cliquez sur l’icône pour générer le code incorporé de votre application.

## Personnalisation de l’application {#section_lvj_xlt_bbb}

Vous pouvez utiliser l’onglet Concepteur d’applications pour sélectionner les options **[!UICONTROL Style]** et **[!UICONTROL Configuration]** pour personnaliser vos applications.

**[!UICONTROL Style]** Les options sont cohérentes pour tous les types d’application ;  **[!UICONTROL Configuration]** Les options varient selon l’application. Le bas du volet **[!UICONTROL Style]** et **[!UICONTROL Configuration]** liste la version de l’application Livefyre pour votre application.

**Sélectionner les options de style**

Mise en forme des applications en fonction de l’identité graphique de votre site. Utilisez les options **[!UICONTROL Style]** pour sélectionner les styles de couleur et de police de l’application.

* **[!UICONTROL Theme]**. Sélectionnez **[!UICONTROL Light]** ou **[!UICONTROL Dark]** pour définir la couleur d’arrière-plan du contenu affiché. Les modifications apportées à ce paramètre sont immédiatement répercutées dans la prévisualisation à droite. Ce paramètre contrôle l’arrière-plan, le texte du contenu, l’utilisateur, l’horodatage, le bouton de partage et la sélection de couleur des icônes de réseau social pour l’application.

* **[!UICONTROL Brand color]**. Sélectionnez une couleur pour votre application. Cette couleur sera utilisée pour les hashtags, URL, noms d’utilisateur, @mentions, graphiques à barres et autres composants spécifiques à l’application, comme affiché dans le volet de prévisualisation à droite.
* **[!UICONTROL Font family]**. Sélectionnez une police pour le contenu généré par l’utilisateur dans votre application. Si vous sélectionnez Georgia ou Times New Roman, vous attribuerez cette police au contenu et Helvetica Neue aux noms d’utilisateur et aux noms d’affichage. La sélection de Helvetica, Verdana ou Trebuchet attribuera ces polices au contenu, aux noms d’utilisateur et aux noms d’affichage.
* **[!UICONTROL Font size]**. Sélectionnez une taille de police par défaut. Ce paramètre aura une incidence sur l’ensemble du contenu et des titres générés par l’utilisateur affichés dans l’application.

## Mise à jour des applications {#section_iwb_4bz_vcb}

Si une version plus récente de l’application existe, un bouton **[!UICONTROL Update]** s’affiche.

Cliquez sur le bouton pour mettre à jour la version la plus récente. Actualisez votre application afin d’utiliser toutes les améliorations et correctifs de fonctionnalités disponibles. Une fois la mise à jour effectuée, les modifications de prévisualisation apportées à votre application (le cas échéant), puis acceptez ou rejetez la mise à jour.

## Ajouter un bouton d’appel à l’action à une application {#section_cpf_cmt_bbb}

Ajoutez un bouton Appel à l&#39;action pour générer des conversions à partir de l&#39;UGC.

Pour ajouter un bouton Appel à l’action à une application :

1. Créez une application Mosaïque ou une application de mur multimédia.

   Pour plus d’informations sur les personnalisations disponibles dans une mosaïque ou un mur multimédia, voir [Personnalisations de mosaïque](/help/using/c-about-apps/c-mosaic-app/c-mosaic-customizations.md) ou [Personnalisations du mur de médias](../c-about-apps/c-media-wall-app/r-media-wall-customizations.md#r_media_wall_customizations).
1. Ouvrez Designer pour l’application.
1. Accédez à la section **[!UICONTROL Call-to-Action Button]** de Designer.
1. Basculez **[!UICONTROL Call-to-Action Button]** sur **[!UICONTROL on]**.

1. Configurez les options du bouton Appel à l&#39;action.

## Prévisualisations d’application {#section_eyy_4mt_bbb}

App Designer vous permet de prévisualisation de votre application dans plusieurs formats de diffusion et d’afficher vos propres données d’exemple ou d’exemple.

Sélectionnez **[!UICONTROL Exit Preview]** pour revenir au mode de conception.

Le mode prévisualisation offre les options suivantes :

* **[!UICONTROL Preview Options]**: Utilisez le menu déroulant pour sélectionner un format de prévisualisation (mobile, tablette ou ordinateur de bureau) et indiquer s’il faut afficher  **[!UICONTROL Sample Data]** ou non les données du flux de l’application.

* 

>[!NOTE]
>
>Si vous désélectionnez **[!UICONTROL Sample Data]** et qu’aucun flux n’est activé pour l’application, aucun contenu ne s’affiche en mode Prévisualisation.

* **[!UICONTROL Share]**: Ouvre l’application dans un nouvel onglet de navigateur et génère une URL qui peut être utilisée pour permettre à des amis, des collègues ou des utilisateurs non-Studio de réviser votre application et de la tester dans différents navigateurs et périphériques.

   >[!NOTE]
   >
   >Tout contenu publié sur une version de révision de votre application sera ajouté à l’application. Si vous ne souhaitez pas que ce contenu apparaisse dans votre application publiée, vous devez le supprimer avant de cliquer sur **Publier**.

* **[!UICONTROL Exit Preview]**: Retourne en mode d&#39;édition.

>[!NOTE]
>
>Le mode prévisualisation est entièrement interactif et vous permet de tester l’expérience utilisateur pour votre application. Si vous activez **[!UICONTROL Allow user posts]**, les utilisateurs disposant du lien de prévisualisation peuvent cliquer sur le bouton **[!UICONTROL What’s on your mind]** et publier le contenu dans votre version de test. Studio liste le contenu tel que publié.

## Publication d’applications {#section_df1_smt_bbb}

App Designer enregistre automatiquement votre application après avoir apporté des modifications. Vous pouvez utiliser **[!UICONTROL Publish]** pour générer un code incorporé ou pour mettre à jour toute version précédemment publiée de cette application avec la version actuelle.

>[!NOTE]
>
>Le fait de cliquer sur **[!UICONTROL Publish]** génère un code incorporé pour l’application et met à jour toutes les instances existantes de l’application si elles ont déjà été publiées.

## Vérifier l&#39;historique des applications {#section_jsk_vmt_bbb}

App Designer enregistre automatiquement votre application une fois que vous avez effectué une modification et génère une nouvelle version pour chaque modification. Vous pouvez utiliser **[!UICONTROL History]** pour ouvrir un panneau répertoriant toutes les versions de votre application. Les versions répertoriées affichent l’éditeur de chaque version et son âge.

Pour consulter l’historique des applications :

1. Cliquez sur une version pour la sélectionner.
1. Actualisez l’affichage de la prévisualisation.
1. Cliquez sur **[!UICONTROL Restore Version]** pour cloner la version sélectionnée et travailler à partir de celle-ci.
1. Cliquez sur le bouton Précédent dans **[!UICONTROL Version History]** pour revenir à votre brouillon de travail.
1. Cliquez sur **[!UICONTROL Share]** pour générer une URL pour la prévisualisation.

>[!NOTE]
>
>Vous ne pouvez pas modifier une application tant que le volet **[!UICONTROL Version History]** est ouvert.

Si un autre éditeur apporte des modifications alors que la fenêtre de votre navigateur est ouverte, Livefyre affiche un message d’avertissement lorsque vous tentez de modifier l’application, vous demandant si vous souhaitez revenir à votre (ancienne) version préliminaire. (Cela empêche plusieurs éditeurs simultanés d’ignorer accidentellement les modifications des uns et des autres.) Cliquez sur **[!UICONTROL Update to Latest]** pour commencer à travailler à partir de la dernière version. Cliquez sur **[!UICONTROL Overwrite]** pour revenir à votre (ancien) état de travail, incrémentez le numéro de version du plus récent et continuez à travailler.

## Onglet Résumé de l’application {#section_rxx_xmt_bbb}

Utilisez l’onglet Résumé de l’application pour gérer les informations relatives à des applications individuelles.

L’onglet Résumé de l’application vous permet d’effectuer les opérations suivantes :

* **[!UICONTROL See on Site]**: Ouvre l’application sur votre site.
* **[!UICONTROL Embed code]**: Disponible uniquement pour les applications publiées, fournit un lien cliquable permettant de générer le code incorporé pour toute application publiée.

   >[!NOTE]
   >
   >Utilisez ce code incorporé pour ajouter des sondages à vos commentaires ou à votre application de blog en direct. Pour plus d’informations, voir Interactions personnalisées > HTML personnalisé.

* **[!UICONTROL View Content]**: Ouvre la page Contenu qui liste tout le contenu publié dans votre application.
* **[!UICONTROL Moderate]**: Ouvre ModQ, qui répertorie tout le contenu en attente de modération pour l’application.
* **[!UICONTROL Developer Info]**: Fournit l’ID de collecte, l’ID d’article et l’ID de site de l’application.

## Incorporer une application sur votre site à l’aide de Studio {#section_w3h_1nt_bbb}

Générez le code incorporé.

Lorsqu’une application est publiée, Livefyre génère et affiche un code incorporé, qui peut être utilisé pour l’incorporer à n’importe quelle page de votre réseau. Pour régénérer le fragment de code HTML utilisé pour incorporer votre application sur une page :

1. Dans l’écran Présentation de l’application, cliquez sur le bouton **[!UICONTROL </> Embed Code]**.
1. Dans le panneau **[!UICONTROL Embed Code]** qui s’ouvre, copiez et collez le fragment de code HTML affiché dans votre page Web.

   Pour les applications nécessitant une authentification utilisateur, votre intégration pour l’authentification utilisateur doit être terminée et située sur la page avant d’ajouter le code incorporé.

## Suppression d’une application {#section_zyx_zyx_xcb}

Vous pouvez supprimer une application de la liste Applications de Livefyre Studio. Lorsque vous supprimez l’application de Studio, celle-ci est masquée de la vue, mais elle ne se supprime ni ne s’arrête :

* L’application ne continue pas à travailler sur un site.
* Flux associés.
* Collections associées dans Livefyre.

Pour supprimer une application :

1. Cliquez sur **[!UICONTROL Apps]**.
1. Cliquez sur les points de suspension à droite du nom de l’application à supprimer.
1. Cliquez sur **[!UICONTROL Delete App]** pour supprimer l’application.
