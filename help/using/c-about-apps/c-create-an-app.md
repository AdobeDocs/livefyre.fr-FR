# Créer une application {#create-an-app}

Pour ajouter une application à votre site, vous devez d’abord créer une application dans Livefyre Studio.

Pour créer une application :

1. Cliquez sur **[!UICONTROL +New App]** dans la page Applications, puis sélectionnez un type d’application dans le menu déroulant.
1. La **[!UICONTROL New App]** boîte de dialogue s’affiche. Entrez un **[!UICONTROL App Title]** nom, sélectionnez un **[!UICONTROL Site]**, puis cliquez **[!UICONTROL Create]** sur pour ouvrir la page Aperçu de l’application.

La page Résumé de l’application comporte 4 onglets :

* **[!UICONTROL App Summary]**: fournit un aperçu du contenu et de l’état de l’application, ainsi que des outils permettant d’accéder à l’application sur son site et de générer son code incorporé.
* **[!UICONTROL Designer]**: vous permet de sélectionner des options de conception pour l’application, notamment des polices, des modèles de couleurs et des options de configuration spécifiques à l’application.
* **[!UICONTROL Streams]**: répertorie toutes les sources de diffusion en continu pour le contenu de l’application. Cliquez sur le titre du flux pour ouvrir la page Détails du flux, à partir de laquelle il peut être modifié.
* **[!UICONTROL Settings]**: vous permet de sélectionner des paramètres spécifiques à l’application, notamment des règles d’ajout de contenu à l’application et les types de contenu à afficher. Cliquez sur l’ **[!UICONTROL Settings]** onglet pour remplacer les paramètres réseau et de site de l’application.

## Liste des applications {#section_eyp_ykt_bbb}

La page de liste Applications donne accès à toutes vos applications Livefyre existantes.

La page de liste Applications affiche les informations suivantes :

* **[!UICONTROL Title]**: Répertorie le titre de votre application et comprend une icône indiquant le type d’application.
* **[!UICONTROL Site]**: Répertorie l’emplacement de l’application, par site.
* **[!UICONTROL Content Count]**: Répertorie le nombre d’éléments de contenu publiés dans l’application.
* **[!UICONTROL Date Modified]**: Répertorie la dernière fois que l’application a été modifiée dans App Designer.
* **[!UICONTROL Embed Code (</>)]**: Cliquez sur l’icône pour générer le code incorporé de votre application.

## Personnalisation de l’application {#section_lvj_xlt_bbb}

Vous pouvez utiliser l’onglet Concepteur d’applications pour sélectionner **[!UICONTROL Style]** et **[!UICONTROL Configuration]** sélectionner des options de personnalisation des applications.

**[!UICONTROL Style]** Les options sont cohérentes pour tous les types d’application ; Les **[!UICONTROL Configuration]** options varient selon l’application. Le bas du **[!UICONTROL Style]** volet et du **[!UICONTROL Configuration]** répertorie la version de l’application Livefyre pour votre application.

**Sélectionner les options de style**

Mettre des applications de style en fonction de la marque de votre site. Utilisez les **[!UICONTROL Style]** options pour sélectionner la couleur et les styles de police de l’application.

* **[!UICONTROL Theme]**. Sélectionnez **[!UICONTROL Light]** ou **[!UICONTROL Dark]** définissez la couleur d’arrière-plan du contenu affiché. Les modifications apportées à ce paramètre sont immédiatement répercutées dans l’aperçu à droite. Ce paramètre contrôle l’arrière-plan, le texte du contenu, le nom d’utilisateur, l’horodatage, le bouton de partage et la sélection de couleur des icônes sociales pour l’application.

* **[!UICONTROL Brand color]**. Sélectionnez une couleur pour votre application. Cette couleur sera utilisée pour les hashtags, URL, noms d’utilisateur, @mentions, graphiques à barres et autres composants spécifiques à l’application, comme indiqué dans le volet d’aperçu à droite.
* **[!UICONTROL Font family]**. Sélectionnez une police pour le contenu généré par l’utilisateur dans votre application. Si vous sélectionnez Georgia ou Times New Roman, vous attribuerez cette police au contenu et Helvetica Neue aux noms d’utilisateur et aux noms d’affichage. Si vous sélectionnez Helvetica, Verdana ou Trebuchet, ces polices seront attribuées au contenu, aux noms d’utilisateur et aux noms d’affichage.
* **[!UICONTROL Font size]**. Sélectionnez une taille de police par défaut. Ce paramètre affectera l’ensemble du contenu généré par l’utilisateur et des titres affichés dans l’application.

## Mise à jour des applications {#section_iwb_4bz_vcb}

S’il existe une version plus récente de l’application, un **[!UICONTROL Update]** bouton s’affiche.

Cliquez sur le bouton pour mettre à jour la version la plus récente. Actualisez votre application afin d’utiliser toutes les améliorations et correctifs de fonctionnalités disponibles. Une fois la mise à jour effectuée, prévisualisez les modifications apportées à votre application (le cas échéant), puis acceptez ou rejetez la mise à jour.

## Ajout d’un bouton d’appel à l’action à une application {#section_cpf_cmt_bbb}

Ajoutez un bouton Appel à l’action pour générer des conversions à partir de l’UGC.

Pour ajouter un bouton Appel à l’action à une application :

1. Créez une application Mosaic ou Media Wall.

   Pour plus d’informations sur les personnalisations disponibles dans un mur de mosaïque ou de médias, voir Personnalisations [de](/help/using/c-about-apps/c-mosaic-app/c-mosaic-customizations.md) mosaïque ou Personnalisations [du mur de](../c-about-apps/c-media-wall-app/r-media-wall-customizations.md#r_media_wall_customizations)médias.
1. Ouvrez Designer pour l’application.
1. Accédez à la **[!UICONTROL Call-to-Action Button]** section dans Designer.
1. Passez **[!UICONTROL Call-to-Action Button]** à **[!UICONTROL on]**.

1. Configurez les options du bouton Appel à l’action.

## Aperçu de l’application {#section_eyy_4mt_bbb}

App Designer vous permet de prévisualiser votre application dans plusieurs formats de diffusion et d’afficher vos propres données ou des exemples.

Sélectionnez **[!UICONTROL Exit Preview]** pour revenir au mode de conception.

Le mode Aperçu offre les options suivantes :

* **[!UICONTROL Preview Options]**: Utilisez le menu déroulant pour sélectionner un format d’aperçu (mobile, tablette ou ordinateur de bureau) et indiquer si vous souhaitez afficher **[!UICONTROL Sample Data]** ou non les données de diffusion en continu de l’application.

* 

>[!NOTE]
>
>Si vous désélectionnez **[!UICONTROL Sample Data]** et qu’aucun flux n’est activé pour l’application, aucun contenu ne s’affiche en mode Aperçu.

* **[!UICONTROL Share]**: Ouvre l’application dans un nouvel onglet de navigateur et génère une URL qui peut être utilisée pour permettre à des amis, des collègues ou des utilisateurs non-Studio de revoir votre application et de la tester dans différents navigateurs et périphériques.

   >[!NOTE]
   >
   >Tout contenu publié dans une version de révision de votre application sera ajouté à l’application. Si vous ne souhaitez pas que ce contenu apparaisse dans votre application publiée, vous devez le supprimer avant de cliquer sur **Publier**.

* **[!UICONTROL Exit Preview]**: Renvoie au mode d’édition.

>[!NOTE]
>
>Le mode Aperçu est entièrement interactif et vous permet de tester l’expérience utilisateur pour votre application. Si vous l’activez **[!UICONTROL Allow user posts]**, les utilisateurs dotés du lien d’aperçu peuvent cliquer sur le **[!UICONTROL What’s on your mind]** bouton et publier du contenu dans votre version de test. Studio répertorie le contenu tel que publié.

## Publication d’applications {#section_df1_smt_bbb}

App Designer enregistre automatiquement votre application une fois que vous avez effectué une modification. Vous pouvez utiliser **[!UICONTROL Publish]** pour générer un code incorporé ou pour mettre à jour toute version précédemment publiée de cette application avec la version actuelle.

>[!NOTE]
>
>Le fait de cliquer sur **[!UICONTROL Publish]** génère un code incorporé pour l’application et met à jour toutes les instances existantes de l’application si elles ont déjà été publiées.

## Vérifier l’historique des applications {#section_jsk_vmt_bbb}

App Designer enregistre automatiquement votre application après avoir apporté une modification et génère une nouvelle version pour chaque modification. Vous pouvez utiliser **[!UICONTROL History]** pour ouvrir un panneau répertoriant toutes les versions de votre application. Les versions répertoriées affichent l’éditeur de chaque version et son âge.

Pour consulter l’historique des applications :

1. Cliquez sur une version pour la sélectionner.
1. Actualisez l’affichage de l’aperçu.
1. Cliquez sur **[!UICONTROL Restore Version]** pour cloner la version sélectionnée et y travailler.
1. Cliquez sur le bouton Précédent dans **[!UICONTROL Version History]** pour revenir à votre brouillon de travail.
1. Cliquez sur **[!UICONTROL Share]** pour générer une URL pour l’aperçu.

>[!NOTE]
>
>Vous ne pouvez pas modifier une application tant que le **[!UICONTROL Version History]** volet est ouvert.

Si un autre éditeur effectue des modifications alors que la fenêtre du navigateur est ouverte, Livefyre affiche un message d’avertissement lorsque vous tentez de modifier l’application, vous demandant si vous souhaitez revenir à votre (ancienne) version préliminaire. (Cela empêche plusieurs éditeurs simultanés d’ignorer accidentellement les modifications de chacun.) Cliquez sur **[!UICONTROL Update to Latest]** pour commencer à travailler à partir de la dernière version. Cliquez sur **[!UICONTROL Overwrite]** pour revenir à l’état de travail (plus ancien), incrémentez le numéro de version du plus récent et continuez à travailler.

## Onglet Résumé de l’application {#section_rxx_xmt_bbb}

Utilisez l’onglet Résumé de l’application pour gérer les informations relatives aux applications individuelles.

L’onglet Résumé de l’application vous permet d’effectuer les opérations suivantes :

* **[!UICONTROL See on Site]**: Ouvre l’application sur votre site.
* **[!UICONTROL Embed code]**: Disponible uniquement pour les applications publiées, fournit un lien cliquable permettant de générer le code incorporé pour toute application publiée.

   >[!NOTE]
   >
   >Utilisez ce code incorporé pour ajouter des sondages à vos commentaires ou à votre application de blog en direct. Pour plus d’informations, voir Interactions personnalisées &gt; HTML personnalisé.

* **[!UICONTROL View Content]**: Ouvre la page Contenu, qui répertorie tout le contenu publié dans votre application.
* **[!UICONTROL Moderate]**: Ouvre ModQ, répertoriant tout le contenu en attente de modération pour l’application.
* **[!UICONTROL Developer Info]**: Fournit l’ID de collection, l’ID d’article et l’ID de site de l’application.

## Incorporer une application sur votre site à l’aide de Studio {#section_w3h_1nt_bbb}

Générez le code incorporé.

Lorsqu’une application est publiée, Livefyre génère et affiche un code incorporé, qui peut être utilisé pour l’intégrer à n’importe quelle page de votre réseau. Pour régénérer le fragment de code HTML utilisé pour incorporer votre application sur une page :

1. Dans l’écran Présentation de l’application, cliquez sur le **[!UICONTROL </> Embed Code]** bouton.
1. Dans le **[!UICONTROL Embed Code]** panneau qui s’ouvre, copiez et collez le fragment de code HTML affiché dans votre page Web.

   Pour les applications nécessitant une authentification utilisateur, votre intégration pour l’authentification utilisateur doit être terminée et située sur la page avant d’ajouter le code incorporé.

## Suppression d’une application {#section_zyx_zyx_xcb}

Vous pouvez supprimer une application de la liste Applications de Livefyre Studio. Lorsque vous supprimez l’application de Studio, l’application est masquée de la vue. Elle ne la supprime pas ni ne s’arrête :

* L’application ne peut plus travailler sur un site.
* Flux associés.
* Collections associées dans Livefyre.

Pour supprimer une application :

1. Cliquez sur **[!UICONTROL Apps]**.
1. Cliquez sur les points de suspension à droite du nom de l’application à supprimer.
1. Cliquez sur **[!UICONTROL Delete App]** pour supprimer l’application.
